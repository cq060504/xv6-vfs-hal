## 一、环境准备

### 1.1 Docker 容器

项目使用 Docker 隔离 RISC-V 和 LoongArch 两个开发环境。

**RISC-V 容器（xv6）**

```bash
# 镜像：debian:bookworm
# 启动并进入容器
sudo docker run -it --rm -v $(pwd):/xv6 debian:bookworm bash

# 容器内安装工具链（首次）
apt-get update && apt-get install -y --no-install-recommends \
    gcc-riscv64-linux-gnu binutils-riscv64-linux-gnu \
    qemu-system-misc qemu-system-data \
    make python3 perl git build-essential dosfstools \
    && rm -rf /var/lib/apt/lists/*

# 后续使用（复用已配置的容器）
sudo docker start xv6 && sudo docker exec -it xv6 bash
```

**LoongArch 容器（xv6-la）**

```bash
# 构建镜像（首次）
cd /xv6/xv6-riscv-xv6-riscv-rev5
sudo docker build -t xv6-loongarch -f Dockerfile.loongarch .

# 启动并进入容器
sudo docker run -it --rm -v $(pwd):/xv6 xv6-loongarch bash

# 后续使用
sudo docker start xv6-la && sudo docker exec -it xv6-la bash
```

### 1.2 关键工具版本

| 工具 | RISC-V (xv6 容器) | LoongArch (xv6-la 容器) |
|------|-------------------|-------------------------|
| **QEMU** | 7.2.22 | 8.2.2 |
| **GCC** | riscv64-linux-gnu-gcc 12.2.0 | loongarch64-linux-gnu-gcc 14.2.0 |
| **基础镜像** | debian:bookworm | ubuntu:24.04 |

### 1.3 重要注意事项

- **两个容器共用同一个源码目录**（通过 `-v $(pwd):/xv6` 挂载）
- **跨架构切换前必须 `make clean`**，否则旧 `.d` 依赖和对象文件会导致编译错误
- **不要两个容器同时 make qemu**（共享 QEMU lock 会冲突）
- 退出 QEMU：`Ctrl-a` 然后按 `x`

## 二、运行测试

### 2.1 RISC-V 启动

```bash
sudo docker start xv6 && sudo docker exec -it xv6 bash
```

进入 docker 后：

```bash
cd /xv6/xv6-riscv-xv6-riscv-rev5
make clean && make qemu
```

### 2.2 LoongArch 启动

```bash
sudo docker start xv6-la && sudo docker exec -it xv6-la bash
```

进入 docker 后：

```bash
cd /xv6/xv6-riscv-xv6-riscv-rev5
make ARCH=loongarch clean && make ARCH=loongarch qemu
```

### 2.3 xv6 原生测试（双架构通用）

在 xv6 shell 中执行：

```bash
usertests -q
```

预期输出 `ALL TESTS PASSED`。

> 注意：测试中可能出现 `usertrap(): unexpected scause ...` 跟着 `OK` 的输出，这是 usertests 的预期负面测试，不算失败。

### 2.4 ext2 测试（双架构通用）

在 xv6 shell 中执行：

```bash
cat demo.sh | sh
```

> 演示流程：自动挂载 ext2 → 运行 test1（跨文件系统拷贝）→ test2（跨FS拷贝+内容验证）→ ext2test（L1~L4 综合功能测试）。

### 2.5 FAT32 测试（仅 RISC-V）

在 RISC-V 的 xv6 shell 中执行：

```bash
fat32test
```

预期输出：

```
fat32test: starting
PASS: mount
PASS: create + write
PASS: lookup + stat
PASS: mkdir
PASS: create nested
PASS: unlink
fat32test: ALL PASSED
```

6 项测试覆盖：挂载(dev=3)、创建文件+写入、重新打开+获取属性、创建子目录、嵌套路径创建、删除文件并验证。

> 注意：LoongArch 暂不支持 FAT32（受 QEMU -bios ROM 加载上限 ~4MB 限制）。
